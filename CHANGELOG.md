# Changelog

All notable changes to **R'a Core** should be documented in this file.

## 2.4.0 - AI-generated task decomposition, auto-continue, skills, and speed

- **AI-generated task decomposition**: Tasks are now generated by the prompt, not hardcoded. Any prompt goes to a parallel tasks section with real task splitting and per-task completion signs.
- **Auto-continue engine**: The system never stops until every task is completed. An intent controller detects pending tasks and automatically continues until all are done, then writes a Final Report.
- **Skills system**: Added `listSkills`, `useSkill`, and `createSkill` tools. Skills are reusable expertise packs stored as markdown files. Relevant skills are auto-injected into the system prompt based on task context.
- **Skill creator**: After solving a novel, repeatable problem, the agent can save it as a skill for future runs.
- **MCP integration**: MCP tools are used in the right scenarios (databases, browsers, issue trackers) with proper domain routing.
- **System prompt rework**: Added speed protocol, task plan protocol, completion protocol, and skills protocol. The system prompt is never broken and works correctly with task planning.
- **Speed optimizations**: Fast workspace context, parallel batch tools, and focused verification for maximum speed.
- **Version bump**: 2.3.0 → 2.4.0

## 1.0.0 - Standalone rewrite

- Rebranded the product to **R'a Core**
- Removed server, shared, and database packages
- Switched to local session, config, auth, and theme persistence under `~/.racore/`
- Added OpenAI/Codex and OpenRouter provider configuration
- Added a dedicated configuration screen
- Added npm onboarding, releases screen, and npm update checking
